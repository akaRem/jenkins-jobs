Creates branches according config.

Based on project infra/release_scripts.

Workflow:

  1. Prepare create_branch.yaml on project infra/release_scripts. (see create_branch/ directory) and push it as change request to master branch;
  2. Run the job with specified CHANGE_BRANCH_CHANGEID parameter (Will be used latest patchset. DRY_RUN will be set as true automatically);
  3. Check results (see log and test results). Correct config if needed;
  4. If results are correct, please merge the change request;
  5. After merge you should run the job with specified MARK_BUILD_BY_TAG and DRY_RUN=false. All the branches will be created and git tag specified on MARK_BUILD_BY_TAG will be created on project infra/release_scripts.
